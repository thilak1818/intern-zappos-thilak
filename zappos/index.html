Reason behind my aspiration for a career in Front-end development

My career as a front-end developer started with Infosys Ltd working for their Banking product Finacle. I was allocated to a project which involved converting the front-end of the product from J2EE architecture to HTML5 with JS components. It was a nuclear team and i was involved in all the stages of front-end development like wireframe designing,User Experience analysis, Conversion coding,new JS component creation,integrating with the existing backend.The project was so interesting and challenging that i hardly felt like taking my mind off it.The project took a year and was then integrated as part of Finacle 11.0 version and this is one of the projects that gave me the satisfaction of adding value to myself and the company and it made me gain insight into a future career as a front-end developer.

My Favourite Projects

Doctor Finder - Web Application

Live Url: www.thilakg.com/projects/edoctor

A web application which helps patients to book appointment with doctors based on their locality. Doctors and patients have their own profile.Doctors provide their availability in their profiles which are then accessed by patients to book slots.

Challenges faced

Creating the time-slots based on the availability provided by the doctor, for the user to select

Changing the style of the booked slots and differentiating it from the available ones

Dynamic profile page view for both doctor and patient i.e a single template page which changes according to the logged in user

Proper session management -patients should never be able to edit doctor's profile

FINUX Front-end conversion project

Converted 120 menus in Liquidity services management module to HTML 5/Javascript Component based architecture. As i said before this is the project that spoke to me about my career as a UI developer.

Proof of Concept -Creating a Treegrid component

Most of the data related to accounts,loans have a hierarchical structure. A grid which shows these data in a user friendly manner with expand collapse option (in a tree format) was needed. A treegrid named FinTreeGrid was proposed to meet this requirement.

Challenges faced

JSON data from service calls had to intelligently organize them into a format accepted by DojoTreeGrid. Also checkbox ,Filter,Sorting has to be integrated into the native Dojo Tree Grid.
